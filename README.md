**Dynamic Report Generator** is a powerful, free, and open-source reporting engine for Laravel applications. It bridges the gap between complex relational databases and non-technical end-users by providing a strictly typed Abstract Syntax Tree (AST) compiler and automated Graph-Based Join resolution.

By leveraging **Virtual Attributes** and **Subquery Pushdown**, this package completely eliminates the "Developer Bottleneck," empowering everyone from floor workers to executives to build complex, self-serve reports without writing a single line of SQL.

---

## 🚀 Key Features

- 🔓 **Data Democratization**: Completely abstracts database architecture (3D Relational Graphs) into simple, 2D checkboxes for your end-users.
- 🧠 **Auto-Join Resolution**: Stop manually writing `LEFT JOIN` clauses. Our Breadth-First Search (BFS) graph algorithm automatically deduces the shortest relationship path between any two Eloquent models.
- ⚡ **O(1) Memory Footprint**: Instead of hydrating thousands of Eloquent models into PHP RAM to run aggregations, Virtual Attributes push the calculations down to the database engine via raw SQL subqueries.
- 🧱 **Strict AST Parsing**: Completely decoupled from your UI framework. The engine accepts strict Data Transfer Objects (DTOs) via JSON payloads, entirely eliminating SQL injection vulnerabilities.
- 💾 **Schema-less Persistence**: Save, load, and execute complex report configurations natively via the database without needing schema migrations.

## 📦 Installation

You can install the package via composer:

```bash
composer require nisalatp/dynamicreportgenerator
```

*(Note: Ensure your `composer.json` is configured to allow minimum stability if you are pulling from a development branch).*

## 🛠 Basic Usage

The package is designed to be interacted with via the `DynamicReport` Facade. It expects a strict JSON payload that translates into a `ReportRequest` AST.

### 1. Generating a Headless Report

```php
use DynamicReport;
use Nisalatp\DynamicReportGenerator\Types\ReportRequest;

// The payload generated by your frontend (Vue, React, AlpineJS)
$ast = new ReportRequest(
    baseModel: 'User',
    targetModels: ['Order', 'Product'],
    selectedAttributes: [ /* ... */ ],
    innerFilters: /* ... */
);

// Generates the optimized Laravel Query Builder instance
$query = DynamicReport::generate($ast);

return response()->json($query->paginate(50));
```

### 2. Registering Virtual Attributes

Administrators can use the Fluent Builder API to attach complex subqueries to models, simplifying the UI for end-users.

```php
use Nisalatp\DynamicReportGenerator\Builders\VirtualAttributeBuilder;

VirtualAttributeBuilder::create('Total Spend')
    ->forBaseModel('User')
    ->withSqlFragment('(SELECT SUM(amount) FROM orders WHERE orders.user_id = users.id)')
    ->register();
```
